My Development Process

First of all, I strongly believe in CLEAN code. I hate copying a code even if it's from my own code from another project. I believe in typing every bit of it except some Bootstrap snippets and rare circumstances. One can only achieve a clean code if he/she got a plan before creating a folder in their computer for the project. These are my basics development process for my projects.

1. Project Birth

This is the phase that the projects idea kicks in. A lot of this phase happens in the mind and a piece of paper. Personally writing something down in paper solve half of my challenges.So I take time and write key components of the idea. I always make sure of exploiting all the features of the project even those I don't know how to implement yet. Doing this have always prove helpful especially when it comes to generating component required structures

2. UI Design

Figma it is. After having the entire project skeleton in plain english on a paper, I always head to my favorite design software, Figma. I take my time here, but it also depends on the size of the project. This very crucial step as it predicts a lot on the number of components you will need, the kind of service the project will need. I believe having the design rigth and realistic is having the real software in advance. I am saying so because all that remains after this step is converting the plain english from step one, and using this design structure to code.

3. Coding

This is the actual coding phase. And with the right design, this phase get a lot easier to implement. I always take my time, implement piece after piece strictly according to the design becuase if you start adjusting ideas at this point, then there was not need for the design and you will end up creating a completely different site. What's important here is that, implementing page after page or from one functionality to the next. After each, polish it to design standard and run neccessary test before going to the next.

4. Polishing | Tests | Deployment | Maintainance

After the actual coding of all the components and functionalities, I always run tests like responsiveness, endpoints, routes, etc. When everything is in place, deployment phase takes action. After that is making sure all functionalities are working as expected. From there, all there is is maintainance and updates where necessary.
